# Jiuge模型动态量化使用说明

## 概述

我们为 jiuge9G4B 模型实现了动态量化功能，可以将模型的线性层权重从 FP16/FP32 动态量化为 INT8，从而减少内存使用并可能提升推理速度。

## 主要特性

- **动态量化**: 在模型加载时将浮点权重动态转换为 INT8
- **选择性量化**: 仅量化线性层（attention和FFN的projection层），保持norm层为浮点
- **兼容性**: 完全兼容原有的推理接口和使用方式
- **可配置**: 支持启用/禁用量化功能

## 文件说明

- `jiuge_quantized.py`: 支持动态量化的主脚本
- `jiuge.py`: 原始的非量化版本脚本

## 使用方法

### 基本用法（启用量化）

```bash
# CPU 推理
python scripts/jiuge_quantized.py --cpu /path/to/jiuge9G4B/model

# GPU 推理
python scripts/jiuge_quantized.py --nvidia /path/to/jiuge9G4B/model

# 多设备推理
python scripts/jiuge_quantized.py --nvidia /path/to/jiuge9G4B/model 2
```

### 禁用量化

如果您想使用新脚本但禁用量化功能：

```bash
python scripts/jiuge_quantized.py --cpu /path/to/jiuge9G4B/model --no-quantization
```

### 支持的设备类型

- `--cpu`: CPU 推理
- `--nvidia`: NVIDIA GPU 推理  
- `--cambricon`: 寒武纪推理
- `--ascend`: 昇腾推理
- `--metax`: MetaX 推理
- `--moore`: Moore 推理
- `--iluvatar`: 天数智芯推理

## 量化详情

### 量化算法

使用对称量化算法将 FP16/FP32 权重转换为 INT8：

1. **计算量化参数**:
   - `scale = (max_val - min_val) / (qmax - qmin)`
   - `zero_point = qmin - min_val / scale`

2. **量化公式**: 
   - `quantized = round(tensor / scale + zero_point).clamp(-128, 127)`

3. **反量化公式**:
   - `dequantized = scale * (quantized - zero_point)`

### 量化层选择

**会被量化的层**:
- Attention 层的 Q、K、V、O projection
- FFN 层的 gate、up、down projection  
- 输出 embedding 层

**不会被量化的层**:
- 输入 embedding 层
- LayerNorm/RMSNorm 层
- bias 参数

### 预期效果

- **内存减少**: 线性层权重内存使用减少约 50%
- **精度影响**: 轻微的精度损失（通常可接受）
- **速度影响**: 在某些硬件上可能获得加速

## 代码示例

### 在代码中使用量化模型

```python
from scripts.jiuge_quantized import JiugeQuantizedForCauslLM, DeviceType

# 创建量化模型
model = JiugeQuantizedForCauslLM(
    model_dir_path="/path/to/jiuge9G4B",
    device=DeviceType.DEVICE_TYPE_CPU,
    ndev=1,
    enable_quantization=True,        # 启用量化
    quantize_linear_layers=True      # 量化线性层
)

# 生成文本
output, avg_time = model.generate(
    input_content="山东最高的山是？",
    max_steps=500,
    temperature_=1.0,
    topk_=1,
    topp_=1.0
)

print(f"生成结果: {output}")
print(f"平均推理时间: {avg_time:.3f}ms")

# 清理资源
model.destroy_model_instance()
```

### 禁用量化

```python
# 创建非量化模型（使用新脚本但禁用量化）
model = JiugeQuantizedForCauslLM(
    model_dir_path="/path/to/jiuge9G4B",
    device=DeviceType.DEVICE_TYPE_CPU,
    ndev=1,
    enable_quantization=False,       # 禁用量化
    quantize_linear_layers=False
)
```

## 注意事项

1. **内存需求**: 量化过程需要额外的临时内存来存储量化参数
2. **精度影响**: INT8 量化会带来一定的精度损失，建议在实际应用前进行验证
3. **硬件支持**: 某些硬件可能对 INT8 运算有更好的优化支持
4. **兼容性**: 量化功能完全兼容原有的推理接口

## 故障排除

### 常见问题

1. **内存不足错误**: 
   - 尝试减少并发请求数量
   - 检查系统可用内存

2. **精度下降过多**:
   - 尝试仅量化部分层
   - 检查模型是否适合量化

3. **速度没有提升**:
   - 某些硬件平台可能不会从 INT8 中获益
   - 考虑使用专门优化的硬件

### 获取帮助

如果遇到问题，请检查：
1. 模型文件是否完整
2. 依赖包是否正确安装
3. 硬件平台是否支持
4. 内存是否充足
